1. Introduction

The objective of this Assigment 2 is the integration of the REST API server based on the Express framework (implemented earlier), together with the Mongoose schema and models to create a full-fledged REST API server.

Three main entities compound the application in focus: DISH, PROMOTION and LEADER.

It may figure out a food business (restaurants, fast-foods, …) where plates are served (dishes), some of them with discounts (promotions), managed by their leaders.

Typical events to organize the inherent activities are CRUD for dishes, promotions and leaders, saved in a Database. Support for insert, access and update comments related to the dishes.



2. Design and Implementation

The implemented REST API is simple, however didatic. The operations uses HTTP requests to GET, PUT, POST and DELETE data to all models representing those entities (Dish, Leader and Promotion). Eventually, the same operations can be applied to the Comments for Dishes.

Each one of these operations has a correspondent event in MongoDB (CRUD). The persistence of the data objects are mapped by Mongoose ODM. Mongoose is a Nodejs library that provides a schema-based solution for modeling your application data. It has a type conversion system, validation, creation of queries and hooks for business logic.

2.1 The REST API Specification

Upon a NodeJS web server, listening port 3000, that’s the list of operations:

  • http://localhost:3000/dishes => GET, POST, DELETE

  • http://localhost:3000/dishes/:dishID => GET, PUT, DELETE

  • http://localhost:3000/leaders => GET, POST, DELETE

  • http://localhost:3000/leaders/:leaderID => GET, PUT, DELETE

  • http://localhost:3000/promotions => GET, POST, DELETE

  • http://localhost:3000/promotions/:promotionID => GET, PUT, DELETE

  • http://localhost:3000/dishes/comments => GET, POST, DELETE

  • http://localhost:3000/dishes/coments/:commentID => GET, PUT, DELETE
